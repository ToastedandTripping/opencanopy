# Razor Review Checklist: OpenCanopy

Accumulated from production bugs. Razor reads this before reviewing any diff.

## Tippecanoe / Tile Build
- **Always use `-L name:file` not `-l name file`** -- `-l` sets a global override; last one wins, all layers merge into one. Found 2026-03-22.
- **Don't use tile-join for multi-layer PMTiles** -- it merges all named layers into one. Use single tippecanoe pass instead. Found 2026-03-22.
- **Verify PMTiles metadata after build** -- inspect `vector_layers` array to confirm all expected layers exist with correct names before uploading/deploying. Found 2026-03-22.

## react-map-gl / MapLibre
- **Never pass `undefined` values in paint props** -- `"fill-outline-color": undefined` breaks MapLibre layer creation. Filter out undefined properties. Found 2026-03-22.
- **Don't pass MapLibre expression arrays where react-map-gl expects scalars** -- interpolation expressions (e.g., `["interpolate", ...]`) in paint properties can break react-map-gl's Layer component. Use static values or layout visibility for show/hide. Found 2026-03-22.
- **Fill layers inside React fragments may fail in react-map-gl** -- fill+outline wrapped in `<>...</>` inside `<Source>` may not register. Standalone line layers in the same Source work fine. Under investigation. Found 2026-03-22.

## Deploy Verification
- **Always verify with DevTools after deploy** -- check: (a) console errors, (b) layer existence via `map.getStyle().layers`, (c) rendered features via `queryRenderedFeatures`, (d) screenshot. "Deploy is live" is not verification. Found 2026-03-22.
- **Check R2/CDN is serving correct file** -- after tile upload, verify range requests return 206 with correct CORS headers. Found 2026-03-22.
